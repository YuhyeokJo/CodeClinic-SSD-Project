# RULE
## 브랜치
### 네이밍: 타입/설명
타입
|타입|뜻|
|-|-|
|feature|새로운 기능 개발 시|
|bugfix|버그 수정 시|
|refactor|리팩토링 시, 기능 변화 없이 코드 구조 변경|
|chore|프로젝트 구조 설계 시, 실제 기능을 바꾸지 않는 작업, 예를 들면 테스트 뼈대만 추가할 때 사용 가능|

설명

케밥case 사용   
ex) feature/add-ssd
## 커밋
### 기준
테스트 코드 통과
### 네이밍: \[타입\] 설명
타입
* \[feature\]: 기능 추가
* \[refactor\]: 기능 변경 없는 리팩토링
